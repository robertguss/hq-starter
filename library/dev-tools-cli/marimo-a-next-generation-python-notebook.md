## The notebook, reinvented

marimo feels like a notebook but is stored as a pure Python program that's Git-friendly, reusable as a module, executable as a script, shareable as an app, reproducible in execution and packaging, and designed for data (with SQL and LLM support built-in).And it's all open source.

### Code and outputs stay in sync

Run one cell and marimo reacts by[running affected cells](https://docs.marimo.io/guides/reactivity/), eliminating the error-prone chore of managing notebook state. For expensive notebooks, configure marimo to[mark outputs as stale](https://docs.marimo.io/guides/expensive_notebooks/)instead of autorunning.

![Image 61: exploring data in marimo](https://cms.marimo.io/landing/reactive.gif)

### Interactive elements

Built-in[interactive elements](https://docs.marimo.io/guides/interactivity/)like sliders, dropdowns,[dataframe tables](https://docs.marimo.io/guides/working_with_data/dataframes/#interactive-dataframes), and[selectable plots](https://docs.marimo.io/guides/working_with_data/plotting/?h=altair#reactive-plots)make working with data feel fast, futuristic, and fun.

![Image 62: exploring data in marimo](https://cms.marimo.io/landing/ui.gif)

### Python-first, Git-friendly

marimo notebooks are[pure Python](https://marimo.io/blog/python-not-json), stored as .py files Version with Git, run as Python scripts, reuse as modules, manage with uv and PEP 723 metadata, test with PyTest, and integrate with your favorite developer tools.

### Reproducible

Notebooks are executed in a deterministic order,[with no hidden state](https://marimo.io/blog/lessons-learned)— delete a cell and marimo deletes its variables while updating affected cells. Optionally[serialize package requirements in notebook files](https://docs.marimo.io/guides/package_management/inlining_dependencies/), with marimo managing virtual environments for you.

### Share as apps, run as scripts

[Share interactive notebooks on the web](https://docs.marimo.io/guides/publishing/)by exporting to WebAssembly-powered HTML or serving notebooks as web apps with the marimo CLI.[Run as scripts](https://docs.marimo.io/guides/scripts/)locally or in pipelines.

### Made for developers

The marimo editor comes with[GitHub Copilot-style autocomplete](https://docs.marimo.io/guides/editor_features/watching/?h=autocom#types-and-autocompletion), hover tooltips,[vim keybindings](https://docs.marimo.io/guides/editor_features/overview/?h=vim+k), code formatting, debugging panels, and[extensive hotkeys](https://docs.marimo.io/guides/editor_features/overview/?h=vim+k). marimo also ships with a[CLI](https://docs.marimo.io/api/cli_args/?h=command+line), a library, and a VS Code extension.

### First-class SQL support

Query dataframes and databases with marimo's[built-in SQL cells](https://docs.marimo.io/guides/working_with_data/). These can integrate with[DuckDB, SQLite, Postgres, MySQL, and many more](https://docs.marimo.io/guides/working_with_data/sql/?h=sql#connecting-to-a-custom-database)

### An AI-native Python editor

Code in an[AI-native editor](https://docs.marimo.io/guides/editor_features/ai_completion/)with copilots, AI assistance, inline edits, and context-aware chat, using any model, hosted or local.
