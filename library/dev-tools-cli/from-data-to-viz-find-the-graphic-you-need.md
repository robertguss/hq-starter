## Caveats

### The best way to visualize data efficiently is probably to avoid the most common mistakes.

From Data to Viz offers you a [gallery of common caveats.](https://www.data-to-viz.com/caveats.html)

[![Image 43](https://www.data-to-viz.com/img/mistake/order_data.png) Order your data When displaying the value of several entities, ordering it makes it way more insightful.](https://www.data-to-viz.com/caveat/order_data.html)

[![Image 44](https://www.data-to-viz.com/img/mistake/cut_y_axis.png) To cut or not to cut? Cutting the Y axis is one of the most controversial practices in data viz. See why.](https://www.data-to-viz.com/caveat/cut_y_axis.html)

[![Image 45](https://www.data-to-viz.com/img/mistake/spaghetti.png) The spaghetti chart A line graph with too many lines becomes unreadable: it is called a spaghetti graph.](https://www.data-to-viz.com/caveat/spaghetti.html)

[![Image 46](https://www.data-to-viz.com/img/mistake/pie.png) Pie chart The human eye is bad at reading angles. See how to replace the most criticized chart ever.](https://www.data-to-viz.com/caveat/pie.html)

[![Image 47](https://www.data-to-viz.com/img/mistake/bin_size.png) Play with histogram bin size Always try different bin sizes when you build an histogram, it can lead to different insights.](https://www.data-to-viz.com/caveat/bin_size.html)

[![Image 48](https://www.data-to-viz.com/img/mistake/boxplot.png) Do boxplots hide information? A boxplot is a great way to summarize a distribution but it hides the sample size and data distribution.](https://www.data-to-viz.com/caveat/boxplot.html)

[![Image 49](https://www.data-to-viz.com/img/mistake/error_bar.png) The problem with error bars Barplots with error bars must be used with great care. Learn why and how to replace them.](https://www.data-to-viz.com/caveat/error_bar.html)

[![Image 50](https://www.data-to-viz.com/img/mistake/multi_distribution.png) Too many distributions If you need to compare the distribution of many variables, don't clutter your graph.](https://www.data-to-viz.com/caveat/multi_distribution.html)

[![Image 51](https://www.data-to-viz.com/img/mistake/overplotting.png) Overplotting Too many points on your scatter plot makes it unreadable? Techniques exist to avoid overplotting.](https://www.data-to-viz.com/caveat/overplotting.html)

[![Image 52](https://www.data-to-viz.com/img/mistake/rainbow.png) The rainbow color palette Avoid the rainbow color palette when you map a numeric variable. So many better palettes exist.](https://www.data-to-viz.com/caveat/rainbow.html)

[![Image 53](https://www.data-to-viz.com/img/mistake/small_multiple.png) Faceting: horizontal or vertical? Placing individual plots horizontally or vertically is an important choice to make.](https://www.data-to-viz.com/caveat/small_multiple.html)

[![Image 54](https://www.data-to-viz.com/img/mistake/counter_intuitive.png) Don't be counter intuitive Your audience is used to a few dataviz standards. Not respecting it can mislead them.](https://www.data-to-viz.com/caveat/counter_intuitive.html)

[![Image 61](https://www.data-to-viz.com/img/section/Violin150.png)](https://www.data-to-viz.com/graph/violin.html)## [Violin](https://www.data-to-viz.com/graph/violin.html)
An alternative to boxplot to compare the distribution of several groups.

About

* * *

Violinplots allow to visualize the distribution of a numeric variable for one or several groups. It is really close from a [boxplot](https://www.data-to-viz.com/caveat/boxplot.html), but allows a deeper understanding of the distribution. 

Violins are particularly adapted when the amount of data is huge and showing individual observations gets impossible.

Common Mistakes

* * *

*   If you have just a few groups, you are probably interested by [ridgeline charts](https://www.data-to-viz.com/graph/ridgeline.html).
*   If you compare groups with very different sample size, [show it](https://www.data-to-viz.com/caveat/boxplot.html).
*   [Ordering groups](https://www.data-to-viz.com/caveat/order_data.html) by median value makes the chart more insightful.

[![Image 62](https://www.data-to-viz.com/img/section/Density150.png)](https://www.data-to-viz.com/graph/density.html)## [Density](https://www.data-to-viz.com/graph/density.html)
Show the density of a numeric variable

About

* * *

A density plot shows the distribution of a numeric variable. It takes only numeric variables as input and is very close from an [histogram](https://www.data-to-viz.com/#histogram). It can be use in the same exact condition.

Common Mistakes

* * *

*   [Play with the bandwith](https://www.data-to-viz.com/caveat/bin_size.html) of your density function.
*   Don't show the distribution of [more than ~5 variables](https://www.data-to-viz.com/caveat/multi_distribution.html). Use Violin or Ridge line plot instead.
*   Avoid filling with [color palettes](https://www.data-to-viz.com/caveat/color_com_nothing.html).

[![Image 63](https://www.data-to-viz.com/img/section/Histogram150.png)](https://www.data-to-viz.com/graph/histogram.html)## [Histogram](https://www.data-to-viz.com/graph/histogram.html)
An accurate graphical representation of the distribution of numeric data.

About

* * *

A histogram takes as input a numeric variable only. The variable is cut into several bins, and the number of observation per bin is represented by the height of the bar. It is possible to represent the distribution of several variable on the same axis using this technique.

Common Mistakes

* * *

*   [Play with the bin size](https://www.data-to-viz.com/caveat/bin_size.html), it can give different insight.
*   Don't show the distribution of [more than ~5 variables](https://www.data-to-viz.com/caveat/multi_distribution.html). Use [violin](https://www.data-to-viz.com/graph/violin.html) or [ridgeline plot](https://www.data-to-viz.com/graph/ridgeline.html) instead.
*   [Avoid filling with color palettes.](https://www.data-to-viz.com/caveat/color_com_nothing.html)

[![Image 64](https://www.data-to-viz.com/img/section/Box1150.png)](https://www.data-to-viz.com/caveat/boxplot.html)## [Boxplot](https://www.data-to-viz.com/caveat/boxplot.html)
Summarize the distribution of numeric variables

About

* * *

A [boxplot](https://www.data-to-viz.com/caveat/boxplot.html) gives a nice summary of one or several numeric variables. The line that divides the box into 2 parts represents the median of the data. The end of the box shows the upper and lower quartiles. The extreme lines show the highest and lowest value excluding outliers.

Common Mistakes

* * *

*   Boxplot hides the sample size of each group, [show it with annotation or box width.](https://www.data-to-viz.com/caveat/boxplot.html)
*   Boxplot [hides the underlying distribution](https://www.data-to-viz.com/caveat/boxplot.html). Use jitter if low number of data points, or use violin with bigger data.
*   [Order your boxplot](https://www.data-to-viz.com/caveat/order_data.html) by median can make it more insightful.

[![Image 65](https://www.data-to-viz.com/img/section/Joyplot150.png)](https://www.data-to-viz.com/graph/ridgeline.html)## [Ridgeline plot](https://www.data-to-viz.com/graph/ridgeline.html)
Several distributions aligned to the same horizontal scale and presented with a slight overlap

About

* * *

A [Ridgeline plot](https://www.data-to-viz.com/graph/ridgeline.html) (sometimes called Joyplot) shows the distribution of a numerical value for several groups. Distribution can be represented using [histograms](https://www.data-to-viz.com/graph/histogram.html) or [density plots](https://www.data-to-viz.com/graph/density.html), all aligned to the same horizontal scale and presented with a slight overlap. 

 It is especially useful when the number of group to represent is high, and thus a classic window separation would take to much space.

Common Mistakes

* * *

*   As with histogram or density plot, [play with bin size](https://www.data-to-viz.com/caveat/bin_size.html) / bandwidth argument.
*   Ridgeline plot works well when there is a clear pattern to discover since it hides a part of the data where the overlap takes place.

The last level (leaves) are located at the extreme outer part of the circle.

It is possible to map a numeric value to each leaf of the graphic, in this case their size will be proportional to this value.

Sunburst diagrams suffer the same issue than [pie](https://www.data-to-viz.com/caveat/pie.html) or [doughnut](https://www.data-to-viz.com/graph/donut.html) plot: it is really hard to compare the size of different slices and thus conveys poorly the information. You should probably consider treemap, circular packing or dendrogram.

Common Mistakes

* * *

*   Do not confound it with a circular stacked barplot.

[![Image 85](https://www.data-to-viz.com/img/section/Line150.png)](https://www.data-to-viz.com/graph/line.html)## [Line chart](https://www.data-to-viz.com/graph/line.html)
Shows the evolution of numeric variables.

About

* * *

A [line chart](https://www.data-to-viz.com/graph/line.html) or line graph displays the evolution of one or several numeric variables. Data points are connected by straight line segments. It is similar to a [scatter plot](https://www.data-to-viz.com/graph/scatter.html) except that the measurement points are ordered (typically by their x-axis value) and joined with straight line segments.

A line chart is often used to visualize a trend in data over intervals of time – a time series – thus the line is often drawn chronologically.

When several values are available for each time stamp, data can be aggregated and a confidence zone is generally added around the trend.

Common Mistakes

* * *

*   [To cut or not to cut](https://www.data-to-viz.com/caveat/cut_y_axis.html) the Y axis?
*   If you need to compare the evolution of 2 different variables, do not use [dual axis](https://www.data-to-viz.com/caveat/dual_axis.html).
*   Mind the [spaghetti chart](https://www.data-to-viz.com/caveat/spaghetti.html): too many lines make the chart unreadable.

A stacked version of the area chart to compare several groups.

About

* * *

A [stacked area chart](https://www.data-to-viz.com/graph/stackedarea.html) is the extension of a basic [area chart](https://www.data-to-viz.com/graph/area.html) which displays the evolution of the value of several groups on the same graphic. The values of each group are displayed on top of each other, what allows to check on the same figure the evolution of both the total of a numeric variable, and the importance of each group.

If only the relative importance of each group interests you, you should probably draw a percent stacked area chart. It is the same thing, but proportion of each group are represented instead of absolute value.

[Stacked area chart](https://www.data-to-viz.com/graph/stackedarea.html) are criticized. Indeed, it can be hard to read the evolution of a specific group, since the reader has to substract the other groups in his mind. It is often better to use small multiple or line plot instead.

*   Use it with care, try using small multiple or line chart instead.
*   The group order (from bottom to top) can have an influence, try several orders.

About

* * *

A [Stream graph](https://www.data-to-viz.com/graph/streamgraph.html) is a type of [stacked area chart](https://www.data-to-viz.com/graph/stackedarea.html). It displays the evolution of a numeric value (Y axis) following another numeric value (X axis). This evolution is represented for several groups, all with a distinct color.

Contrary to a stacked area, there is no corner: edges are rounded what gives this nice impression of flow. Moreover, areas are usually displaced around a central axis, resulting in a flowing and organic shape.

[Stream chart](https://www.data-to-viz.com/graph/streamgraph.html) gets really useful when displayed in an interactive mode: highlighting a group gives you directly an insight of its evolution.

*   For a more accurate but less attractive figure, consider a [line chart](https://www.data-to-viz.com/graph/line.html) or using small multiple.
