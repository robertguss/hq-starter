## Why specs when you have "plan" mode?

The real question is about scope.

Cursor and Claude Code work great for small changes. But for anything larger, you're constantly re-prompting, course-correcting, and babysitting.

The bigger the feature, the more time you spend keeping AI on track.

Shotgun plans the entire feature upfront—then splits it into stages.

Each stage is what Cursor would call "one plan." You get 5 neat PRs instead of one 10k-line monster nobody will review.

AI agents derail on large features—they forget context, rebuild existing stuff, or go off-spec halfway through.

Think of it as "here are 8 stages with blueprints for each" instead of "build me a house."

## What's in a spec?

*   01 Codebase research

Understands your existing patterns and architecture 
*   02 Implementation stages

Breaks features into reviewable PR-sized chunks 
*   03 File-by-file instructions

Exact changes for each file in each stage 
*   04 Agent-ready output

Feed directly to Cursor, Claude Code, or Codex
