### What does 16x Prompt do exactly?

▼
16x Prompt is a desktop application that helps you compose prompts from various inputs. It generates prompts from the task instructions, source code files and the formatting instructions. ▼

Yes, 16x Prompt supports dark mode. You can switch between light and dark mode from the top right corner of the application. The application will remember your choice and open in the selected mode next time.

### I am getting connection error when using API. What should I do?

▼

1. Check the logs from

16x Prompt

to find the error message. 

For macOS:

~/Library/Logs/16x Prompt/main.log

For Windows:

%USERPROFILE%\AppData\Roaming\16x Prompt\logs\main.log

For Linux:

~/.config/16x Prompt/logs/main.log

2. If you see the

unable to get local issuer certificate
error, chances are you are using a corporate self-signed certificate. You can set the environment variable

NODE_TLS_REJECT_UNAUTHORIZED=0
when running the application to bypass the certificate check. 

For example, on macOS, run

NODE_TLS_REJECT_UNAUTHORIZED=0 open /Applications/16x\ Prompt.app
to open the application.

3. If you are behind a proxy or VPN, you can try to disable it and see if the connection works. You can also try to use a different network to see if the connection works.

If you are still facing issues, feel free to

[contact us](https://prompt.16x.engineer/contact).

### I am getting installation error DiscardVirtualMemory on Windows 8.

▼
You may encounter "The procedure entry point DiscardVirtualMemory could not be located in the dynamic link library" error on Windows 8. This is because the underlying framework for the app no longer supports Windows 8, in line with[end of support for Windows 8 from Microsoft](https://support.microsoft.com/en-us/windows/windows-8-1-support-ended-on-january-10-2023-3cfd4cde-f611-496a-8057-923fba401e93). You can try to upgrade to Windows 10 / Windows 11 or use a different machine with Windows 10 / Windows 11. ### 16x Prompt is not opening on Windows or macOS. What should I do?

▼
You can try to clear the application data and restart the application. On Windows, go to%APPDATA% and delete the 16x Prompt folder. On macOS, go to~/Library/Application Support and delete the 16x Prompt folder. If you are still facing issues, feel free to[contact us](https://prompt.16x.engineer/contact).

### 16x Prompt is not opening on Linux with "The SUID sandbox helper binary was found, but is not configured correctly" error.

▼

This is due to a[bug](https://github.com/electron/electron/issues/42510#issuecomment-2171583086)with Electron on Ubuntu 24 LTS. You can refer to this[workaround](https://github.com/electron/electron/issues/41066#issuecomment-1913509942)which fixes the error by executing

sudo sysctl -w kernel.apparmor_restrict_unprivileged_userns=0
and disabling the kernel.apparmor_restrict_unprivileged_userns. If you are still facing issues, feel free to[contact us](https://prompt.16x.engineer/contact).
