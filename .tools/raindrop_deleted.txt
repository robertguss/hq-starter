# Raindrop deletion ledger — tombstone file.
#
# One raindrop_id per line. Anything after `#` is a comment; blank lines
# are ignored. `raindrop_import.py` reads this on every run and skips
# listed IDs alongside raindrops already present in library/.
#
# Use case: you delete a library/<...>.md file because the bookmark isn't
# worth keeping, but the raindrop still exists in your Raindrop.io account.
# Without this ledger, the next `raindrop_import.py` run would re-create
# the file. Add the raindrop_id here to make the deletion permanent on
# the vault side without also having to delete the raindrop upstream.
#
# To find a raindrop_id: grep the (now-deleted) file, or look in the
# Raindrop web UI URL when viewing the bookmark.
#
# Example:
#   1641398064  # InstaSDR — SPA, content not scrapable
#   937833116   # Nomic AI — SPA, content not scrapable
